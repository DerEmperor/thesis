\subsubsection{Sources}
\label{subsubsec:sources}

The sources define the origin for the data. Currently we are using Git (GitHub and GitLab), Confluence and Jira as source. For Example from GitHub we pull among other commits, issues and pipelines.

\subsubsection{Infrastructure}
\label{subsubsec:infrastructure}

The infrastructure provides the database for the project. The data is stored in a data lake, which is a repository, where data is stored as-is without further structure \autocite{datalake_aws}. Each set of data passes through 4 Stages: raw, bronze, silver, gold.

\subsubsection{Pipeline}
\label{subsubsec:pipeline}

The pipelines transform the data between the stages.


\subsubsection{Analytics}
\label{subsubsec:analytics}

In the last step the data is used to provide multiple analytics for customers. Examples for such analytics are:

\begin{itemize}
	\item Calculating the Costs of Inner Source Collaboration by Computing the Time Worked \autocite{analytic_cost}
	\item Estimating Software Contribution Time	from Git and GitHub Metadata \autocite{analytic_contribution}
\end{itemize}

\subsubsection{Services}
\label{subsubsec:services}

Services are helper modules for pipelines and analytics. Such helper modules are:

\begin{itemize}
	\item AI: a framework for communicating with a \ac{LLM}.
	\item config handling: Parses configuration files for pipelines.
\end{itemize}

\subsection{Data Flow}
\label{subsec:dataflow}

After pulling the source the raw data is stored exactly as provided as JSON. In the data lake the Data is processed multiple times and the states are called bronze, silver and gold. The processing is done by the pipelines.

 Each pipeline hast 3 steps:
 
 \begin{enumerate}
 	\item Read the data from the data lake
 	\item transform data
 	\item write the transformed data to the data lake
 \end{enumerate}

Figure \ref{fig:dataflow} displays an overview of all components. \autocite{github_mecois}

\subsubsection{JSON to Bronze}
\label{subsubsec:json2bronze}

The \texttt{JSONtoBronzePipeline} processes the raw JSON data provided by the sources and writes it unchanged to the Data lake. The data itself isn't changed in any way. Each source has an own table.

\subsubsection{Bronze to Silver}
\label{subsubsec:bronze2silver}

The \texttt{BronzetoSilverPipeline} processes the bronze data in multiple steps to be ready for the analytics.

\paragraph{Flatten table} 

The bronze state cannot represent nested data. (e.g. a GitHub commit contains an author and the author itself has multiple properties). In the first step those nested data in columns is split in multiple columns, so that each on only contains a single data point. 

\paragraph{Drop irrelevant columns}

Not all data is useful. Some columns (e.g. the URL to the avatar of a GitHub user) are dropped. 

\paragraph{Fix dates}

Dates are still stored as String. This steps casts these Strings into 

\paragraph{Anonymize Data}

The data is sent to the anonymization service for processing.

\paragraph{add UUID}

Each table has a way to create an \ac{UUID} by concatenating some columns and hashing the result.

\subsubsection{Silver to Silver}
\label{subsubsec:silver2silver}

Identities may update in the identity management system. The \texttt{UpdateIdentityPipeline} updates any identities in the data lake that have been updated in the SortingHat.

\subsubsection{Silver to Gold}
\label{subsubsec:silver2gold}

There is no single \texttt{Silver2GoldPipeline}. The analytics represent the gold to silver pipelines. Each analytic uses the silver data to provide some metric.

\subsubsection{Gold to Gold}
\label{subsubsec:gold2gold}

Some analytics may need some mechanism to update the generated metric. This process is defined as gold to gold transformation.

\section{Identity Management System}
\label{sec:identity_management_system}

The identity management system is responsible for storing all information about individuals. The project is using the tool SortingHat from the tool set GrimoireLab by CHAOSS. This application is provided under the GPL-3.0 license.

One individual my have multiple identities on different platforms. The system can pool these different accounts into one individual. These individuals can also be grouped into teams. Multiple teams can form an organization.  \autocite{sortinghat} \autocite{sortinghat_repo}
